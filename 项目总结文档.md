# 视觉引导自动摘钩系统 — 项目总结

## 系统架构

```
YOLO + D455 → target_pole (camera_link)
  ↓ TF 链: base_link → flange → camera_link → target_pole
target_tracker_node: 计算 base_link 坐标 + 偏移
  ↓ PointStamped → /abb4600_unhook/manual_target
手动目标状态机 (FSM)
  ↓ safe_tcp_base_mm / guiji2_tail
Action Server (IK + 轨迹 + 执行)
  ↓ FollowJointTrajectory
ABB IRB4600 机械臂
```

## 工作空间

| 路径 | 用途 | 编译命令 |
|------|------|---------|
| `~/xm/abb4600_rws_ws` | 机械臂运动控制 | `source /opt/ros/jazzy/setup.bash && colcon build` |
| `~/ros2_hook_ws` | 视觉识别 | `source .venv_ros2/bin/activate && source /opt/ros/jazzy/setup.bash && colcon build` |

两者都是 ROS2 Jazzy。视觉端需要 venv（Python 依赖），机械臂端不需要。

```bash
# ⑥ 结束
ros2 topic pub -1 /abb4600_unhook/skip_return_home std_msgs/msg/Empty "{}"
```

## 状态机流程

```
IDLE
  ↓ start
WAIT_OPERATOR_CONFIRM
  ↓ confirm_start
WAIT_MANUAL_TARGET  ← 视觉自动发目标
  ↓ 收到坐标
MOVING_TO_TARGET    (IK → 单点轨迹 → 关节移动)
  ↓ 完成
WAIT_MANUAL_VERIFY  ← 等待 start_guiji2_tail
  ↓ start_guiji2_tail
EXECUTING_PRESET_UNHOOK_PATH  (12段 guiji2 + delta)
  ↓ 完成
WAIT_MANUAL_VERIFY  ← 等待 unhook_success
  ↓ unhook_success
WAIT_RETURN_HOME
  ↓ skip_return_home
DONE
```

## 全部命令速查

| 命令 | 作用 |
|------|------|
| `reset` | 任意状态复位到 IDLE |
| `start` | IDLE → WAIT_OPERATOR_CONFIRM |
| `confirm_start` | 确认执行，视觉自动发目标 |
| `start_guiji2_tail` | 执行 12 段 guiji2 摘钩轨迹 |
| `unhook_success` | 确认摘钩成功 |
| `unhook_failed` | 摘钩失败，退回重新接收目标 |
| `skip_return_home` | 结束，不回安全点 |
| `return_home` | 回安全点 |
| `cancel` | 紧急取消 → ERROR |
| `pause` | 暂停机械臂 |
| `resume` | 恢复 |

## 关键改动总结

### 1. Humble → Jazzy 迁移
- 所有 18 个包重新编译为 Jazzy
- `plan.trajectory_` → `plan.trajectory`（MoveIt API 变化）
- `computeCartesianPath` 去掉 `jump_threshold` 参数
- SMACC2 改为可选依赖

### 2. 绕过 MoveGroupInterface
- Jazzy 下 `move_group` 不提供 `get_planning_scene` 服务
- MoveGroupInterface 构造函数因此卡死
- 改为自己加载 RobotModel + 直接 IK + 直接发轨迹给控制器
- IK 使用 robot_model_loader + setFromIK

### 3. 轨迹格式
- 使用单点轨迹（和回 p10 命令一致）
- velocity=0，让控制器自己从当前状态插值
- 时长按关节位移和速度比例计算

### 4. 摘钩轨迹 (guiji2_tail)
- old_anchor = (1526.8, 815.0, 1112.0)mm，硬编码
- new_anchor = 视觉目标
- delta = new_anchor_tool0 - old_anchor_tool0
- 12 段: p710 + delta → p630 + delta → ... → p590 + delta
- 每段独立 IK + 单点轨迹执行

### 5. 控制器容差 (abb_controllers.yaml)
```yaml
constraints:
  stopped_velocity_tolerance: 1.0  # 原 0.1
  trajectory: 6.28                 # 原 1.0 (6.28=2π，基本不拦截)
  goal: 0.5                        # 原 0.05
```

### 6. 速度
- `manual_target_unhook_state_machine_node.cpp` 第 255 行：`velocity_scale = 0.50`

### 7. XYZ 偏移
- 唯一修改处：T5 命令行 `offset_x:=0.08 offset_y:=-0.09 offset_z:=0.08`
- 代码默认值为 0，不改命令行就不加偏移

### 8. 视觉 YOLO 竖直约束
- `yolo_tf_node.py` 第 222 行：`max_dx_base = 0.032`（32mm）
- 两点在 base_link 下 X 偏差超过此值则过滤
